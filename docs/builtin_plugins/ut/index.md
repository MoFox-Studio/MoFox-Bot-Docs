# 实用命令（utility_commands）

实用命令集合插件，收纳一组常用的运维 / 管理类命令。

## 概述

`utility_commands` 当前提供 `/清空上下文` 命令，用于在不重启机器人或删除数据库记录的前提下，清空聊天流的对话上下文。

## 命令清单

### /清空上下文

清空聊天流的对话上下文。

- **触发词**：`/清空上下文` 或 `/clearctx`（中英文均可）
- **权限**：仅 `OWNER` 可用
- **效果**：清空后，机器人将不再加载该时间点之前的历史消息，效果在重启后依然生效。**消息记录仍保留在数据库中，不会被删除。**

#### 用法

| 命令 | 作用 |
|------|------|
| `/清空上下文` | 清空当前聊天流的上下文 |
| `/清空上下文 群` | 清空所有群聊上下文 |
| `/清空上下文 群 <群号>` | 清空指定群的上下文 |
| `/清空上下文 私` | 清空所有私聊上下文 |
| `/清空上下文 私 <QQ号>` | 清空指定私聊的上下文 |
| `/清空上下文 all` / `/清空上下文 全部` | 清空所有聊天流的上下文 |
| `/clearctx [群\|私\|all\|全部] [...]` | 英文别名 |

#### 使用场景

- 机器人对话「记忆错乱」或上下文累积过长导致回复质量下降时
- 想让机器人重新开始一段对话，但不删除历史消息记录
- 测试新 prompt 时清除旧上下文

#### 示例

```
/清空上下文
/清空上下文 群 123456789
/清空上下文 私 987654321
/清空上下文 all
```

## 注意事项

- 仅 `OWNER` 级别用户可用
- 清空操作不可撤销，请确认后再执行
- 清空的是**内存上下文**与「该时间点之前的历史消息加载点」，数据库中的消息记录不会被删除
