## 项目概述
MoFox 一键包是一个集成了Python环境的多服务机器人管理工具，专为简化机器人系统的部署、运行和维护而设计。该程序提供了友好的命令行界面，支持一键启动、更新和管理多个相关服务。

## 系统要求
- 操作系统：Windows（支持PowerShell和CMD）
- Git：用于仓库管理和更新
- 稳定的网络连接以进行仓库更新

## 安装与环境准备
1. 解压 `OneKey-Plus.zip` 到任意目录，**请勿直接在压缩包内运行程序**。
2. 本项目自带Python环境，无需额外安装Python。
3. 确保系统已安装Git，打开命令行执行：
   ```bash
   git --version
   ```

## 快速启动
- 进入解压后的目录，双击运行 `启动一键包程序.bat`，程序将自动检测环境并启动主程序。
- 也可以使用命令行执行：
  ```bash
  python onekey.py
  ```

## 首次使用配置
- 第一次运行时，程序会自动执行更新和配置向导。
- 如果未完成初始化，程序会自动执行更新程序和配置向导，完成后生成初始化标记。
- 建议首次启动后，选择菜单中的：
  - 选项13：安装所有依赖包
  - 选项12：更新所有仓库到最新版本
- 启动程序后，访问 Napcat 的 WebUI，点击“网络配置”，新建一个 WebSocket 客户端，将端口号从默认的 8082 修改为 8095，保存并启动。

## 功能说明
- **服务管理**：支持启动、停止和监控多个机器人服务，包括MoFox主程序及其适配器等。
- **版本控制与更新**：集成GitHub私有仓库自动更新，支持Token认证和依赖自动安装。
- **系统维护**：自动安装依赖、修复权限问题、显示系统信息。
- **服务组合**：
  - QQ机器人组合：MoFox主程序 + 相关适配器 + 服务

## 常见问题
- **pip安装失败？**
  选择菜单选项18进行权限修复，或以管理员身份运行程序。
- **仓库更新失败？**
  检查网络连接和Git配置，确保有仓库访问权限。
- **服务启动失败？**
  检查对应服务目录和文件是否存在，查看系统信息确认环境配置。

## 安装脚本
本一键包内置了安装脚本，解压后可以运行提供的批处理文件 `启动更新程序.bat`，自动完成更新和依赖安装。

## 安装建议
- 建议以管理员权限运行安装和更新脚本，避免权限问题。
- 更新过程中请确保网络连接稳定，防止更新失败。
- 使用包内自带的Python环境，避免与系统其他Python版本冲突。
