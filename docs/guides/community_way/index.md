# 社区部署方式

欢迎来到 MoFox-Core 社区部署方式专区！这里汇集了由社区成员贡献的各种便捷部署方案。

## 什么是社区部署方式？

社区部署方式是指由 MoFox-Core 社区成员开发和维护的、旨在简化 MoFox-Core 部署流程的工具和脚本。这些工具通常针对特定的使用场景或平台进行了优化，可以让你更快速、更轻松地部署 MoFox-Core。

## 特点

- 🚀 **快速部署** - 自动化的安装脚本，减少手动配置步骤
- 🛠️ **社区维护** - 由热心的社区成员开发和维护
- 🎯 **场景优化** - 针对特定使用场景（如 Armbian、特定 Linux 发行版等）进行优化
- 📝 **详细文档** - 配有详细的使用说明和配置指南

## 需要帮助？

如果在使用社区部署方式时遇到问题：

1. 首先查看对应工具的详细文档
2. 检查工具维护者提供的联系方式（通常在文档中注明）
3. 加入 [MoFox-Bot QQ 群](https://qm.qq.com/q/jfeu7Dq7VS) 寻求帮助
4. 在 GitHub 上提交 Issue

## 与官方部署方式的区别

| 特性 | 官方部署方式 | 社区部署方式 |
|------|------------|------------|
| 维护方 | MoFox-Core 官方团队 | 社区成员 |
| 通用性 | 适用于所有平台 | 通常针对特定平台优化 |
| 更新频率 | 随官方版本更新 | 由维护者决定 |
| 技术支持 | 官方支持 | 社区支持 |
| 自动化程度 | 需要一定手动配置 | 通常更加自动化 |

## 选择建议

- **新手用户**: 推荐使用社区部署方式中的自动化安装脚本，可以大大降低部署难度
- **进阶用户**: 可以根据自己的需求选择合适的部署方式，或参考社区方案自行定制
- **特殊平台**: 如果你使用的是特定的硬件平台（如 ARM 设备），优先查看是否有对应的社区部署方案
