## 安装插件

::: danger 重要提示
在安装任何插件之前，请**务必**仔细阅读插件的说明文档（通常为 `README.md` 文件/帖子说明）！这将帮助你了解：
- 插件的具体功能和使用方法
- 必要的配置步骤
- 依赖项要求
- 注意事项和已知问题
:::

安装插件的过程非常简单，我们提供多种方式：

### 方式一：下载插件包（推荐）

1. **加入 Kook 社区**：点击 [邀请链接](https://kook.vip/RQoR4v) 加入官方服务器
2. **找到插件发布频道**：进入「插件发布专区」→「插件发布」
3. **下载插件文件**：在频道中找到你需要的插件，下载对应的 `.mfp` 文件或压缩包
4. **安装插件**：
   - 将下载的文件放入 MoFox Bot 的 `plugins` 目录
   - 如果是压缩包，需要先解压
   - 如果是 `.mfp` 文件，可以直接放置（Bot 会自动识别）
5. **配置插件**：根据插件说明文档进行配置

### 方式二：通过 Git Clone

如果插件提供了 Git 仓库地址，你也可以通过 `git clone` 的方式安装：

1. **获取仓库地址**：在 Kook 插件发布频道或插件说明中查找 Git 仓库地址
2. **进入插件目录**：
   ```bash
   cd /path/to/MoFox-Bot/plugins
   ```
3. **克隆仓库**：
   ```bash
   git clone <插件的 Git 仓库地址>
   ```
4. **安装依赖**（如果需要）：
   ```bash
   cd <插件目录>
   uv pip install -r requirements.txt  # 如果有的话
   ```
   :::tip 注意
   大部分情况下Neo-MoFox会自动安装插件依赖，但如果插件说明里提到了额外的 Python 包，还是建议手动安装一下，避免运行时出问题。
   :::
5. **配置插件**：根据插件说明文档进行配置

## 重启 Bot

无论是通过哪种方式安装，为了让新插件生效，你都需要**重启 Neo-MoFox**。

## 验证安装

重启后，如何确认插件是否成功加载了呢？很简单，去查看日志！

在 Neo-MoFox 的 `logs` 文件夹中，找到最新的日志文件，打开它。如果你看到类似下面这样的日志，就说明插件已经成功加载了：

```log
[21:58:25] plugin_manager | INFO | ✅ 插件 'xxxxxx' 的组件注册完成
[21:58:25] plugin_manager | INFO | ✅ 插件加载成功: xxxxx vX.X.X
```

恭喜你，现在你已经成功安装了一个新的插件！快去体验一下它带来的新功能吧！

## 常见问题

### 如何找到特定的插件？

在 Kook 服务器的「插件发布」频道中，你可以：
- 使用 Kook 的搜索功能搜索插件名称
- 浏览频道消息，查看最新发布的插件
- 在「语言专区」或其他频道询问是否有满足你需求的插件

### 插件下载后无法加载怎么办？

1. **检查文件位置**：确保插件文件放在正确的 `plugins` 目录下
2. **检查文件完整性**：确认下载的文件没有损坏
3. **查看日志**：检查 `logs` 目录下的日志文件，查找错误信息
4. **检查依赖**：确认是否安装了插件所需的 Python 依赖包
5. **寻求帮助**：在 Kook 社区的相关频道询问，或联系插件作者

### 如何更新已安装的插件？

- **通过 Git 安装的插件**：在插件目录下执行 `git pull`
- **通过文件下载的插件**：重新下载最新版本，覆盖旧文件
- **记得重启 Bot**：更新后需要重启 Bot 以使更改生效
