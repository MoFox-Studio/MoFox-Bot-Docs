# 插件安装指南

欢迎来到 Neo-MoFox 插件的世界！在这里，你可以找到各种有趣的插件来扩展 Neo-MoFox 的功能。

## 插件获取途径

### Neo-MoFox 插件市场（推荐）

现在推荐优先使用 **Neo-MoFox 插件市场** 获取插件：

[打开 Neo-MoFox 插件市场](https://39.96.71.162/)

插件市场可以帮你：

- 🔍 **搜索插件**：按插件名称、作者、标签查找插件
- 📦 **下载插件包**：在插件详情页直接下载最新版本或指定版本
- 🔔 **订阅插件更新**：订阅后可在 **「我的」** 页面集中查看，并可配合访问令牌让 Neo-MoFox 同步订阅
- 🧩 **查看插件依赖**：确认插件是否依赖其他前置插件
- 💬 **查看评论与评分**：参考其他用户的安装体验

::: danger 安全提醒
插件市场中的插件由作者自行上传。安装前请确认插件名称、作者、简介、版本、依赖和评论。只安装你信任来源的插件。
:::

## 安装前必读

::: danger 重要提示
在安装任何插件之前，请**务必**仔细阅读插件详情页、插件说明文档或作者发布说明！这将帮助你了解：

- 插件的具体功能和使用方法
- 必要的配置步骤
- 依赖项要求
- 注意事项和已知问题
- 插件是否适配你当前的 Neo-MoFox 版本
:::

## 使用插件市场安装插件

### 第一步：打开插件市场

1. 打开浏览器，进入 [Neo-MoFox 插件市场](https://39.96.71.162/)。
2. 如果是第一次进入插件市场，会先看到 **「首次使用安全提示」** 弹窗。
3. 仔细阅读提示后，点击右下角的 **「我已知悉」**。
4. 随后可能会出现 **「更便利·更轮椅」** 的功能提示弹窗，点击右下角的 **「知道了，不再提示」**。
5. 进入插件市场首页后，就可以开始查找插件了。

### 第二步：查找插件

你可以用下面几种方式找到插件：

- 在顶部搜索框输入插件名称、作者或标签，然后点击 **「搜索」**。
- 点击顶部导航的 **「推荐」**，查看推荐插件。
- 点击顶部导航的 **「浏览」**，按分类浏览插件。
- 如果已经拿到插件详情页链接，可以直接在浏览器打开。

进入插件详情页后，请先确认这些信息：

- 插件标题是否是你要安装的插件
- 插件 ID 是否正确
- 作者是否可信
- 最新版本号和更新时间是否合理
- 插件简介、依赖、评论和评分是否符合预期

### 第三步：找到安装面板

插件详情页右侧会有一个 **「安装」** 面板。这个面板通常会显示：

- 最新稳定版
- 发布时间
- 累计下载
- 订阅人数
- 评分
- **「下载插件」** 按钮
- **「订阅更新」** 按钮

如果你的浏览器窗口比较窄，右侧 **「安装」** 面板可能会移动到页面下方。向下滚动就能看到。

### 第四步：下载插件

安装最新稳定版：

1. 在 **「安装」** 面板点击 **「下载插件」**。
2. 浏览器会打开或下载插件包。
3. 如果浏览器只是打开了下载链接，请按浏览器提示保存文件。
4. 下载完成后，保留插件包，继续下一步。

安装指定版本：

1. 在插件详情页点击 **「版本」** 标签。
2. 找到你需要的版本。
3. 点击该版本右侧的 **「下载」**。
4. 下载完成后，保留插件包，继续下一步。

### 第五步：放入插件目录

将下载到的插件包放入 Neo-MoFox 的插件目录。通常路径是：

```text
<Neo-MoFox安装目录>/plugins/
```

常见处理方式：

- 如果下载的是 `.mfp` 插件包，可以直接放入 `plugins` 目录。
- 如果下载的是压缩包，请按插件说明处理；需要解压时，解压后的插件目录应放在 `plugins` 目录下。
- 不要把插件放到 `config`、`data`、`logs` 等目录里。

### 第六步：订阅更新（推荐）

回到插件市场的插件详情页，在 **「安装」** 面板点击 **「订阅更新」**。按钮状态变成 **「已订阅」** 后，说明订阅成功。

订阅成功后，可以点击插件市场顶部导航栏的 **「我的」**，在 **「我的订阅」** 中集中查看已订阅插件。

### 第七步：配置访问令牌（用于同步订阅）

如果你希望 Neo-MoFox 主程序主动从插件市场同步你的订阅，需要在插件市场生成访问令牌：

1. 登录插件市场。
2. 点击顶部导航栏的 **「我的」**。
3. 找到 **「插件市场访问令牌」** 区域。
4. 如果还没有令牌，点击 **「生成令牌」**。
5. 页面会显示 **「新令牌」**，点击 **「复制令牌」**。
6. 回到 Neo-MoFox，按主程序或 WebUI 的提示把这个令牌填入插件市场同步配置。

::: warning 令牌只在生成时完整显示
请在生成后立刻复制并妥善保存。重新生成令牌会让旧令牌立即失效；点击 **「撤销」** 后，Neo-MoFox 将无法继续使用该令牌从市场同步订阅。
:::

::: tip 哪些情况不需要令牌？
只有**已经安装到本地的 `.mfp` 插件**，并且该插件已经在插件市场注册，Neo-MoFox 才能在不配置访问令牌的情况下处理更新检查。想要同步你的市场订阅列表、自动下载订阅插件，仍然需要配置访问令牌。
:::

## 通过 Git Clone 安装插件（开发者 / 特殊情况）

如果插件作者提供了 Git 仓库地址，你也可以通过 `git clone` 的方式安装：

1. **获取仓库地址**：在插件市场、插件详情页或插件说明中查找 Git 仓库地址。
2. **进入插件目录**：

```bash
cd /path/to/Neo-MoFox/plugins
```

3. **克隆仓库**：

```bash
git clone <插件的 Git 仓库地址>
```

4. **安装依赖**（如果需要）：

```bash
cd <插件目录>
uv pip install -r requirements.txt  # 如果有的话
```

:::tip 注意
大部分情况下 Neo-MoFox 会自动安装插件依赖。如果插件说明里提到了额外的 Python 包，按插件作者说明处理即可。
:::

5. **配置插件**：根据插件说明文档进行配置。

## 重启 Neo-MoFox

无论通过哪种方式安装，为了让新插件生效，你都需要**重启 Neo-MoFox**。

## 验证安装

重启后，如何确认插件是否成功加载？很简单，查看日志。

在 Neo-MoFox 的 `logs` 文件夹中，找到最新的日志文件并打开。如果你看到类似下面这样的日志，就说明插件已经成功加载：

```log
[21:58:25] plugin_manager | INFO | ✅ 插件 'xxxxxx' 的组件注册完成
[21:58:25] plugin_manager | INFO | ✅ 插件加载成功: xxxxx vX.X.X
```

恭喜你，现在你已经成功安装了一个新的插件！快去体验一下它带来的新功能吧！

## 更新插件

### 方式一：订阅并配置访问令牌（推荐）

1. 打开插件市场中的插件详情页。
2. 在右侧 **「安装」** 面板点击 **「订阅更新」**。
3. 点击顶部导航栏的 **「我的」**。
4. 在 **「插件市场访问令牌」** 区域点击 **「生成令牌」** 或 **「重新生成令牌」**。
5. 点击 **「复制令牌」**。
6. 回到 Neo-MoFox，按主程序或 WebUI 的提示把令牌填入插件市场同步配置。
7. Neo-MoFox 后续即可同步订阅列表，并处理订阅插件的下载与更新。

### 方式二：手动下载新版本

1. 打开插件市场中的插件详情页。
2. 查看右侧 **「安装」** 面板中的最新稳定版。
3. 点击 **「下载插件」** 下载新版本。
4. 将新版本插件包放入 `plugins` 目录，按提示覆盖或替换旧版本。
5. 重启 Neo-MoFox。

### 方式三：Git 更新

如果你是通过 Git Clone 安装的插件，可以进入插件目录执行：

```bash
git pull
```

更新后同样需要重启 Neo-MoFox。

## 常见问题

### 如何找到特定的插件？

推荐先在插件市场查找：

- 使用顶部搜索框搜索插件名称、作者或标签。
- 点击 **「推荐」** 查看推荐插件。
- 点击 **「浏览」** 按分类查找。
- 在插件详情页查看依赖、评论、评分和版本信息。

如果插件市场找不到，可以联系插件作者确认是否已经上架，或查看插件详情页中提供的主页、仓库、发布页等外部链接。

### 插件下载后无法加载怎么办？

1. **检查文件位置**：确保插件文件放在正确的 `plugins` 目录下。
2. **检查文件完整性**：确认下载的文件没有损坏。
3. **查看日志**：检查 `logs` 目录下的日志文件，查找错误信息。
4. **检查依赖**：确认是否安装了插件所需的 Python 依赖包。
5. **检查版本兼容性**：确认插件支持你当前的 Neo-MoFox 版本。
6. **寻求帮助**：在插件市场评论区询问，或联系插件作者。

### 点击“下载插件”后没有反应怎么办？

1. 检查浏览器是否拦截了弹窗或下载。
2. 尝试切换到插件详情页的 **「版本」** 标签，点击具体版本右侧的 **「下载」**。
3. 换一个浏览器重试。
4. 如果仍然失败，可以去插件作者提供的 GitHub Release 或其他发布页下载。

### 订阅更新后还需要配置 access token 吗？

如果只是更新**已经安装的 `.mfp` 插件**，并且这个插件已经在插件市场注册，Neo-MoFox 可以不依赖 access token 进行更新检查。

如果你希望 Neo-MoFox 同步你的市场订阅列表、自动下载订阅插件，仍然需要配置 access token。配置方法：进入插件市场顶部导航栏 **「我的」** → **「插件市场访问令牌」** → **「生成令牌」** → **「复制令牌」**，再按 Neo-MoFox 主程序或 WebUI 的提示填写。
